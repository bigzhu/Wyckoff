
### 细节说明

#### 第一阶段：阅读
- 用户按照章节顺序阅读 markdown 文件

- 我根据每一节的核心要点向你提问
- 问题形式：概念理解、案例分析、实战应用等

- 用自己的语言回答问题

- 不理解的地方可以坦诚说出

- 评价你的回答（理解对的地方、不足的地方）

- 强调重点、细节、实战要点

#### 第五阶段：记录
- 将整个问答过程记录到"测试"部分

## 🔑 核心概念速查

### 三大工具
- **价格** - 蜡烛的长度、高低点位
- **成交量** - 成交量的高度、变化速度
- **走势速度** - 上涨或下跌的斜率

### 三大原则
- **供求关系原则** - 需求>供应=上涨，供应>需求=下跌
- **因果关系原则** - 因（准备过程）→ 果（趋势形成）
- **努力和结果原则** - 努力≠结果=停止行为=反转信号

### 关键术语
| 术语 | 含义 |
|------|------|
| CM | 聪明钱/主力机构 |
| 超卖 | 价格持续急速下跌，无反弹 |
| 恐慌抛售(SC) | 下跌中的潮水般抛售 |
| 停止行为 | 努力没有产生相应结果 |
| 垂直需求柱 | 长阳+大成交量 |
| 垂直供应柱 | 长阴+大成交量 |
| 吸筹 | CM的建仓过程 |
| 派发 | CM的出货过程 |

## 💡 学习建议

### 阅读建议
1. **逐节深入** - 不要贪快，每节都要真正理解
2. **多看案例** - 书中的图表很重要，多看几遍
3. **对照原文** - 在 wyckoff_content 中查看原始内容
4. **做笔记** - 用自己的语言总结核心思想

### 提问建议
- 对概念的疑惑：「什么是...？」「为什么...？」
- 对案例的理解：「图1-3中...是什么意思？」
- 对应用的困惑：「在实盘中怎样识别...？」
- 对关系的理解：「A和B的区别是什么？」

### 答疑记录建议
- **标题** - 简洁，能反映问题核心
- **问题** - 完整的用户提问
- **解答** - 清晰的分层讲解
- **关键点** - 重要概念、要点突出
- **实战应用** - 如何在真实交易中应用

### 测试记录建议
- **我的问题** - 针对核心要点的检验问题
- **你的回答** - 完整的用户回答
- **我的评价和补充**：
  - ✅ 理解正确的部分
  - ⚠️ 需要补充的部分（我的补充讲解）
  - 📌 关键要点（强调重点、细节、实战要点）

### "答疑"和"测试"的区别与使用场景

| 维度 | 答疑 | 测试 |
|------|------|------|
| **发起者** | 用户提问 | 我提问 |
| **目的** | 解答疑惑 | 检验理解 |
| **适用场景** | 遇到不懂的地方 | 阅读完一个知识点后 |
| **形式** | Q&A（用户问我答） | Q&A（我问用户答） |
| **记录价值** | 澄清误解、补充细节 | 检验学习效果、深化理解 |

### 使用流程

#### 答疑区的使用
1. ❓ 你在阅读过程中遇到不理解的地方
2. 💬 你向我提问（任何形式）
3. 📝 我做出解答并记录到"答疑区"

**例子**：
> 用户：「什么是停止行为？」
> Claude：给出详细解答
> 记录：在答疑区记录这个 Q&A

#### 测试区的使用
1. 📖 你完整阅读了一个知识点
2. 🧠 我基于这个知识点提出检验问题
3. 💬 你用自己的语言回答
4. 🎓 我评价回答并进行补充讲解
5. 📝 记录到"测试区"

**例子**：
> Claude：「请用自己的语言解释什么是'背景'，以及为什么它在5步看盘法中最重要？」
> 用户：给出回答
> Claude：评价和补充讲解
> 记录：在测试区记录这个完整的问答过程
